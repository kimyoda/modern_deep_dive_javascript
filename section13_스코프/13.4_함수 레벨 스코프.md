# 13.4 함수 레벨 스코프

- 코드 블록이 아닌 함수에 의해서 지역스코프가 생성된다.
- 대부분의 프로그래밍 언어는 모든 코드 블록이 지역 스코프를 만든다. 이를 **블록 스코프**라 한다.
- **var 키워드로 선언된 변수는 오로지 함수의 코드 블록(함수 몸체)만을 지역 스코프로 인정**한다.
- 이를 **함수 레벨 스코프라 한다.**

예제 13-7

```js
var x = 1;

if (true) {
  // var 키워드로 변수는 함수의 코드 블록(함수 몸체)만을 지역 스코프로 인정한다.
  // 함수 밖에서 var 키워드로 선언된 변수는 코드 블록 내에서 선언되도 모두 전역 변수다.
  // x는 전역 변수다. 이미 선언된 전역 변수 x가 있어 x 변수는 중복 선언된다.
  var x = 10;
}

console.log(x); // 10
```

- 전역 변수 x 선언, if 문 코드 블록 내 x 변수 선언
- if문 코드 블록 전역 변수다.
- var 키워드 변수는 함수 레벨 스코프만 인정, 함수 밖에서 var 키워드로 선언된 변수는 모두 전역 변수다.
- 전역 변수 x는 중복 선언되고 결과 전역 변수의 ㄱ밧이 재할당된다.

예제 13-8

```js
var i = 10;

// for  문에서 선언한 i는 전역 변수다. 이미 선언된 전역 변수 i가 있어 중복 선언된다.
for (var i = 0; i < 5; i++) {
  console.log(i); // 0 1 2 3 4
}
// 의도치 않게 변수 값 변경
console.log(i); // 5
```

- 블록 레벨 스코프는 for 문에서 반복을 위해 for 문의 코드 블록 내에서 유효한 지역 변수다.
- var 키워드로 선언된 변수는 블록 레벨 스코프를 인정하지 않아 i 변수는 전역변수가 된다.
- 그 결과 의도치 않은 전역 변수의 값이 재할당된다.
- var 키워드로 선언된 변수는 오로지 함수의 코드 블록만을 지역 스코프로 인정한다.
- ES6에서 도입된 let, const 키워드는 블록 레벨 스코프를 지원한다.

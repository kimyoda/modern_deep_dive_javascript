# 15.1 var 키워드로 선언한 변수의 문제점

- ES5까지 변수를 선언할 수 있는 방법은 var 키워드를 사용하는거였다.
- 다음과 같은 특징이 있다

---

## 15.1.1 변수 중복 선언 허용

- var 키워드로 선언한 변수는 중복 선언이 가능하다.
  예제 15-1

```js
var x = 1;
var y = 1;

// var 키워드로 선언된 변수는 같은 스코프 내에 중복 선언을 허용한다.
// 초기화문이 있는 변수 선언문은 엔진에 의해 var 키워드가 없는 것처럼 작동한다.
var x = 100;
// 초기화문이 없는 변수 선언문은 무시된다.
var y;

console.log(x); // 100
console.log(y); // 1
```

- x, y 변수는 중복 선언되었다.
- 중복 선언하면 초기화문 유무에 따라 다르게 동작한다.
- 초기화문이 있는 변수 선언문은 `var`가 없는 것처럼 동작한다
- 초기화문이 없는 변수 선언문은 무시된다.(에러 발생 x)
- 변수를 중복 선언하여 값까지 중복 할당하면 먼저 선언된 변수의 값이 변경된다.

---

## 15.1.2 함수 레벨 스코프

- var 키워드로 선언한 변수는 함수의 코드 블록만을 지역스코프로 인정한다.
- 함수 외부에서 var 키워드로 선언한 변수는 코드 블록 내에서 선언해도 전역 변수가 된다.
  예제 15-2

```js
var x = 1;

if (true) {
  // x는 전역 변수다. 선언된 전역 변수 x는 중복 선언된다.
  // 의도치 않게 변수 값이 변경되는 부작용을 발생시킨다.
  var x = 10;
}

console.log(x); // 10
```

- for문의 변수 선언문에서 var키워드로 선언한 변수도 전역 변수가 된다.
  예제 15-3

```js
var i = 10;

// for문에서 선언한 i는 전역 변수다. 선언된 전역 변수 i가 있어 중복 선언이다.
for (var i = 0; i < 5; i++) {
  console.log(i); // 1 2 3 4 5
}

// 의도치 않은 i 변수의 값의 변경이다.
console.log(i); // 5
```

- 함수 레벨 스코프는 전역 변수를 남발할 가능성이 높다.
- 의도치 않게 전역변수가 중복 선언되는 경우의 발생이다.

---

## 15.1.3 변수 호이스팅

- var 키워드로 변수를 선언, 변수 호이스팅에 의해 변수 선언문이 스코프의 선두로 끌어 올려진 것처럼 동작한다.
- 변수 호이스팅에 의해 var 키워드로 선언한 변수는 선언문 이전에 참조할 수 있다.
- 할당문 이전에 변수를 참조하면 undefined를 반환한다.

예제 15-4

```js
// 변수 호이스팅에 의해 foo변수가 선언. (1. 선언 단계)
// 변수 foo는 undefined로 초기화된다. (2. 초기화 단계)
console.log(foo); // undefined

// 변수 값 핧당(3. 할당 단계)
foo = 123;

console.log(foo); // 123

// 변수 선언문은 런타임 이전에 자스 엔진에 의해 암죽적으로 실행된다.
var foo;
```

- 변수 선언문 이전에 변수를 참조하는 것은 변수 호이스팅에 의해 에러 발생 x
- 가독성을 떨어뜨리고 오류를 발생시킬 여지를 남긴다.

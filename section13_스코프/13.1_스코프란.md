# 13.1 스코프란?

---

- 스코프(scope, 유효범위)는 자바스크립트를 포함한 모든 프로그래밍 언어의 기본적이고 중요한 개념이다.
- var 키워드로 선언한 변수, let or const 키워드로 선언한 변수의 스코프는 다르게 동작한다.
- 스코프는 변수, 함수와 관련이 있다.

- 함수의 매개변수는 몸체 내부에서만 참조할 수 있고 몸체 외부에서 참조할 수 없다.
- 매개변수를 참조할 수 있는 유효범위, 매개변수의 스코프가 몸체 내부로 한정되기 떄문이다.

예제 13-1

```js
function add(x, y) {
  // 매개변수는 함수 몸체 내부에서만 참조할 수 있다.
  // 즉, 매개변수의 스코프(유효범위)는 함수 몸체 내부다.
  console.log(x, y); // 2 5
  return x + y;
}

add(2, 5);

// 매개변수는 함수 몸체 내부에서만 촘자할 수 있다.
console.log(x, y); //ReferenceError: x is not defined
```

- 변수는 코드 가장 바깥 영역뿐 아니라 코드 블록이나 함수 몸체 내에서 선언할 수 있다.
- 코드 블록이나 함수는 중첩될 수 있다.

예제 13-2

```js
var var1 = 1; // 코드의 가장 바깥 영역에서 선언한 변수

if (true) {
  var var2 = 2; // 코드 블록 내에서 선언한 하수
  if (true) {
    var var3 = 3; // 중첩된 코드 블록 내에서 선언한 함수
  }
}

function foo() {
  var var4 = 4; // 함수 내에서 선언한 변수

  function bar() {
    var var5 = 5; // 중첩된 함수 내에서 선언한 변수
  }
}

console.log(var1); // 1
console.log(var2); // 2
console.log(var3); // 3
console.log(var4); // ReferenceError: var4 is not defined
console.log(var5); // ReferenceError: var5 is not defined
```

- 변수는 자신이 선언된 위치에 의해 자신이 유효한 범위, 코드가 변수 자신을 참조할 수 있는 범위가 결정된다.
- 변수 뿐만 아니라 모든 식별자가 해당한다.
- **모든 식별자(변수 이름, 함수 이름, 클래스 이름 등)은 자신이 선언된 위치에 의해 다른 코드가 식별자 자신을 참조할 수 있는 유효 범위가 결정된다.**
- 스코프는 식별자가 유효한 범위를 말한다.

예제 13-3

```js
var x = "global";

function foo() {
  var x = "local";
  console.log(x); // 1
}

foo();

console.log(x); // 2
```

- 코드의 가장 바깥 영역과 foo 함수 내부에 같은 이름을 갖는 x변수를 선언, 1과 2에서 x 변수를 참조한다.
- 이때 엔진은 두 개의 변수 중 어떤 변수를 참조해야 하는 지 결정한다.
- 이를 **식별자 결정**이라 한다.
- 스코프를 통해 어떤 변수를 참조해야 할 것인지 결정한다.
- 스코프란 자바스크립트 엔진이 **식별자를 검색할 때 사용하는 규칙**이라고 할 수 있다.

- 자바스크립트 엔진은 코드를 실행할 때 코드의 문맥을 고려한다.

---

⭐ 코드의 문맥과 환경

> 💡 **코드가 어디서 실행되며 주변에 어떤 코드가 있는지**를 렉시컬 환경이라고 부른다.  
> 즉, 코드의 문맥은 렉시컬 환경으로 이뤄진다.  
> 이를 구현한 것이 **실행 컨텍스트(execution context)**이며,  
> 모든 코드는 실행 컨텍스트에서 평가되고 실행된다.  
> 스코프는 실행 컨텍스트와 깊은 관련이 있다.

---

⭐ var 키워드로 선언한 변수의 중복 선언

> 💡 `var` 키워드로 선언된 변수는 **같은 스코프 내에서 중복 선언이 허용**된다.  
> 이는 의도치 않게 변수값이 재할당되어 변경되는 부작용을 발생시킨다.

### 예제 13-4

```js
function foo() {
  var x = 1;
  // var 키워드로 선언된 변수는 같은 스코프 내에서 중복 선언을 허용한다.
  // 아래 변수 선언문은 자바스크립트 엔진에 의해 var 키워드가 없는 것처럼 동작한다.
  var x = 2;
  console.log(x); // 2
}

foo();
```

- 하지만 let이나 const 키워드로 선언된 변수는 같은 스코프 내에서 중복 선언을 허용하지 않는다.

```js
function bar() {
  let x = 1;
  // let이나 const 키워드로 선언된 변수는 같은 스코프 내에서 중복 선언을 허용하지 않는다.
  let x = 2; // SyntaxError: Identifier 'x' has already been declared
}

bar();
```

---

### 📖 **1. 스코프(Scope)란?**

- 스코프(scope)는 변수, 함수, 클래스 등의 **식별자가 참조 가능한 범위**를 의미한다.
- 프로그래밍 언어에서 스코프는 **변수 이름의 충돌을 방지**하고, 변수의 가시성을 관리하는 개념이다.

자스에서는 스코프를 통해 어떤 변수를 참조해야 하는 지 결정한다.

이 과정을 식별자 결정이라고 하고, 자스 엔진은 코드를 실행할 때 변수의 유효 범위를 스코프를 기준으로 검색한다.

---

### 📖 2. 스코프의 종류

자스 스코프는 크게 전역 스코프와 지역 스코프로 나뉜다.

1. 전역스코프: 코드의 가장 바깥 영역에서 선언된 변수는 어디서든 참조 가능
2. 지역 스코프: 함수나 코드 블록 내부에서 선언된 변수는 해당 영역 내부에서 참조 가능

---

### 📖 3. 스코프와 변수 참조 예제

```jsx
var x = "global";

function foo() {
  var x = "local";
  console.log(x); // 'local'
}

foo();

console.log(x); // 'global'
```

→ 전역 변수 x와 함수 내부 지역 변수 x는 서로 다른 스코프에 존재한다.

`foo` **함수 내부에서 참조하는 `x` 는 지역 스코프의 변수**이고, **전역에서 참조하는 `x` 는 전역 스코프의 변수**이다.

---

### 📖 4. 스코프와 식별자 결정

자스 엔진은 변수를 참조할 때, 어떤 변수를 사용할 지 스코프를 기준으로 결정한다.

→ 식별자 결정이라고 부르고, 엔진은 스코프 체인을 따라 변수를 검색한다.

---

### 📖 5. 실행 컨텍스트와 스코프의 관계

자스는 코드가 실행될 때 실행 컨텍스트를 생성한다.

실행 컨텍스트는 **변수, 함수 선언, 스코프 정보 등을 담고 있는 객체**, 모든 코드는 실행 컨텍스트에서 평가되고 실행된다.

> 🔔 실행컨텍스트는 렉시컬 환경을 기반으로 만들어지고, 스코프의 구현 메커니즘과 관련이 있다.

---

### 📖 6. var, let, const의 스코프 차이

자스에서 변수를 선언할 때 사용하는 키워드(var, let, const)에 따라 스코프 동작이 다르다.

### 📖 7. 스코프의 네임스페이스 역할

스코프는 네임스페이스 역할을 한다. 같은 이름의 변수를 서로 다른 스코프에서 사용할 수 있도록 하여 변수 충돌을 방지한다.

---

### 📖 8. var 키워드의 문제점: 중복 선언 허용

`var` 키워드로 선언된 변수는 같은 스코프 내에서 중복 선언이 허용된다.

변수 값이 재할당되어 변경되는 부작용이 발생할 수 있다.

```jsx
function foo() {
  var x = 1;
  var x = 2; // 중복 선언 허용
  console.log(x); // 2
}

foo();
```

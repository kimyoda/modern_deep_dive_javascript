# 15.3 const 키워드

- const 키워드는 상수를 선언하기 위해 사용한다.
- 반드시 상수만을 위해 사용하지 않는다.
- const 키워드의 특징은 let과 대부분 동일, 다른점을 중심으로 살펴본다.

---

## 15.3.1 선언과 초기화

- const 키워드로 선언한 변수는 선언과 동시에 초기화가 된다.
  예제 15-13

```js
const foo = 1;
```

- 그렇지 않으면 에러가 발생한다.
  예제 15-14

```js
const foo; // SyntaxError: Missing initializer in const declaration
```

- const 키워드로 선언한 변수는 let과 마찬가지로 블록 레벨 스코프를 가진다.
- 변수 호이스팅이 발생하지 않는것처럼 동작한다.
  예제 15-15

```js
{
  // 변수 호이스팅이 발생하지 않은것처럼 동작한다
  console.log(foo); // ReferenceeError
  const foo = 1;
  console.log(foo); // 1
}

console.log(foo); // ReferenceError: foo is not defined
```

---

## 15.3.2 재할당 금지

- var 또는 let 키워드로 선언한 변수는 재할당이 자유롭다.
- const 키워드로 선언한 변수는 재할당이 금지된다.
  예제 15-16

```js
const foo = 1;
foo = 2; // TypeError
```

---

## 15.3.3 상수

- const 키워드로 선언한 변수에 원시 값을 할당한 경우 변수 값을 변경할 수 없다.
- 원시 값은 변경 불가능한 값(immutable value)이므로 재할당 없이 값을 변경할 수 있는 방법이 없다.
- const 키워드를 상수로 표현하는데 사용한다.

- 변수의 상대 개념인 **상수는 재할당이 금지된 변수를 말한다.**
- 상수도 값을 지정하기 위한 메모리 공간이 필요해 변수라고 할 수 있다.
- 상수는 재할당이 금지된다.
- 상태 유지, 가독성, 유지보수의 편의를 위해 적극적으로 사용한다.
  예제 15-17

```js
// 세전 가격
let preTaxPrice = 100;

// 세후 가격
// 0.1의미를 명확히 알기 어려워 가독성이 좋지않다.
let afterTaxPrice = preTaxPrice + preTaxPrice * 0.1;

console.log(afterTaxPrice); // 110
```

---

## 15.3.4 const키워드와 객체

- const 키워드로 선언된 변수에 원시 값을 할당한 경우 변경할 수 없다.
- 하지만 **const로 선언된 변수에 객체를 할당한 경우 값을 변경할 수 있다.**
- 변경 가능한 값인 객체는 재할당 없이도 직접 변경이 가능하다.
  예제 15-18

```js
const person = {
  name: "Kim",
};

// 객체는 변경 가능한 값이다. 재할당 없이 변경이 가능하다.
person.name = "Lee";

console.log(person); // {name: "Lee"}
```

- const는 **재할당을 금지할 뿐 "불변"을 의미하지 않는다.**
- 프로퍼티 동적 생성, 삭제, 프로퍼티 값의 변경을 통해 객체를 변경하는 것은 가능하다.
- 객체가 변경되더라도 변수에 할당된 참조 값은 변경되지 않는다.
